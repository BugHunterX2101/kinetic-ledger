Applies authenticate and authorise middleware. Middleware JWT verification (authenticate.ts) and role-guard factory (authorise.ts). Attaches decoded user to req.user. Controller Layer Validates request input via Zod / Pydantic schema. Calls service. Serialises response envelope. Service Layer Implements business logic: role checks, soft-delete guard, dashboard aggregations. Calls repository methods. Repository Layer All SQL / ORM queries live here. Returns plain domain objects вҖ” no HTTP types leak in. Data Layer PostgreSQL with two core tables: users and financial_records. Optional audit_log table. 2.2 Authentication Flow Step Description 1. Login POST /auth/login accepts { email, password }. Bcrypt compares hash. On success, signs JWT (HS256 / RS256). 2. JWT Payload { sub: userId, role: &apos;admin&apos; | &apos;analyst&apos; | &apos;viewer&apos;, iat, exp } 3. Per-Request Bearer token in Authorization header. authenticate middleware verifies and attaches req.user. 4. Role Guard authorise([&apos;admin&apos;]) factory returns middleware that rejects 403 if req.user.role is not in the allowed list. 5. Inactive User If is_active = false, login returns 403 immediately regardless of password correctness. 3. Project File Structure The project adopts a module-first layout (also called feature folders). An optional audit_log table provides an append-only mutation trail if implemented. 4.1 users Field Type Notes id UUID Primary key, auto-generated (gen_random_uuid()) email VARCHAR(255) Unique, not null вҖ” used as login identifier password_hash TEXT bcrypt hash (cost 12); never returned in API responses role ENUM &apos;viewer&apos; | &apos;analyst&apos; | &apos;admin&apos; вҖ” exactly one per user is_active BOOLEAN Default true. false = soft-deactivated; blocked at login created_at TIMESTAMPTZ Set automatically on INSERT; not updatable 4.2 financial_records Field Type Notes id UUID Primary key, auto-generated amount DECIMAL(12,2) Required, non-zero; stored as signed decimal type ENUM &apos;income&apos; | &apos;expense&apos; category VARCHAR(100) Free-form string (e.g. Salary, Rent, Travel) date DATE ISO 8601; defaults to CURRENT_DATE on insert notes TEXT Optional narrative description is_deleted BOOLEAN Soft-delete flag; default false. Role assignment and changes are Admin-only operations. The JWT payload carries the role claim; it is validated on every protected request without a database lookup. Action Viewer Analyst Admin View dashboard summary вң“ вң“ вң“ View financial records вң“ вң“ вң“ Filter / search records вң— вң“ вң“ Create financial records вң— вң— вң“ Update financial records вң— вң— вң“ Soft-delete records вң— вң— вң“ Manage users (create/update) вң— вң— вң“ Assign roles вң— вң— вң“ Access trend analytics вң— вң“ вң“ 6.1 Role Escalation Policy Only Admins may call PATCH /users/:id to change a user&apos;s role. An Admin cannot demote themselves below admin (prevents lockout). Role changes take effect on the next login (JWT is re-issued on login only). 7. Validation &amp; Error Handling Input is validated at the Controller layer before any service logic executes.
